# 诺亚核心 · 规则体系

> 7条铁律 — 记忆系统的宪法

## 铁律

### 1. 温度索引必须更新
每次访问诺亚核心中任何文件后，必须立即更新 `_温度索引.md` 的最后访问日期和访问次数。忘写则温度记忆系统失效。**此为最高优先级工作流程规则。**

### 2. 动态文件不等归档
`_进度.md`、`_对接全记录.md`、`_经验技能包.md`、`_外部资源索引.md` 项目进行中就要记录，不能拖到归档阶段。

### 3. 归档文件不提前写
`_项目描述.md`、`_里程碑.md` 仅在项目正式完成时生成。进行中不碰这两个文件。

### 4. 不确定就问GCat
一句话涉及多个项目或分不清归属时，直接问GCat，不猜测。

### 5. 规则落地补丁
设计方案经GCat确认并执行后，必须扫描设计方案文件，将其中所有可执行规则/约束/阈值提取到 `_记忆执行规则.md` 并同步更新永久记忆。不可留在设计方案文件中不落地。

### 6. 文件删除检查
删除任何文件前，先确认是否有唯一副本。无副本则不可删除。如已误删，立即从对话历史或备份恢复。

### 7. 自指闭环
本规则文件和 `_温度索引.md`、`_记忆规则.cfg.md` 作为诺亚核心记忆管理系统的一部分，自身也受温度生命周期约束。记忆系统不给自己开特权。

## 对话扫描规则

每10~15轮对话（或感觉项目相关内容出现时），强制扫描近期对话：

1. 解析每句话 → 匹配到对应项目暂存
2. 无匹配（闲聊/噪音）→ 丢弃
3. 模糊归属 → 问GCat
4. 批量更新对应项目的动态文件
5. 更新温度索引

## 更新频率

| 更新类型 | 频率 | 触发者 |
|---------|------|-------|
| 实时追加 | 每次对话 | 执行Agent |
| 每日巡检 | 每天一次 | cronjob |
| 归档生成 | 项目结束 | 执行Agent |
| 温度迁移 | 每天一次 | cronjob |
